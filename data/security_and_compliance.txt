NOVAMED HEALTH - SECURITY & COMPLIANCE DOCUMENTATION
====================================================

DATA SECURITY
-------------
NovaMed Health is committed to protecting patient data and organizational data at all levels.

ENCRYPTION:
- All data at rest is encrypted using AES-256 encryption
- All data in transit is encrypted using TLS 1.2 and TLS 1.3
- Database backups are encrypted end-to-end
- Encryption keys are managed using AWS KMS (Key Management Service)

ACCESS CONTROL:
- Role-Based Access Control (RBAC) is implemented across all modules
- Multi-Factor Authentication (MFA) is mandatory for all admin accounts
- MFA is optional but recommended for standard user accounts
- Single Sign-On (SSO) is supported via SAML 2.0 and OAuth 2.0
- Privileged Access Management (PAM) for internal engineering access

INFRASTRUCTURE:
- Hosted on AWS (ap-south-1 primary region, ap-southeast-1 DR region)
- Virtual Private Cloud (VPC) isolation per customer for enterprise tier
- Web Application Firewall (WAF) deployed at edge
- DDoS protection via AWS Shield Standard
- Intrusion Detection System (IDS) enabled on all production servers

COMPLIANCE CERTIFICATIONS:
- ISO 27001:2022 certified (audit completed December 2023)
- SOC 2 Type II audit completed (March 2024, report available under NDA)
- DPDP Act 2023 (India Digital Personal Data Protection Act) compliant
- HIPAA-aligned controls implemented (for international customers)
- ABDM (Ayushman Bharat Digital Mission) compliant and registered

VULNERABILITY MANAGEMENT:
- Penetration testing conducted annually by third-party (Cobalt.io)
- Automated vulnerability scanning via Snyk and AWS Inspector
- Bug bounty program active on HackerOne (launched Q2 2024)
- Security patches applied within 72 hours for critical CVEs

INCIDENT RESPONSE AND BREACH NOTIFICATION:
NovaMed Health follows a structured 5-phase incident response process: Detection, Containment, Eradication, Recovery, and Post-Incident Review.

Detection and Monitoring:
- 24/7 security monitoring via SIEM (Splunk) with automated alerting
- Anomaly detection triggers automatic incident ticket creation
- On-call security engineer responds within 15 minutes of alert

Containment and Notification Timeline:
- 0-1 hour: Incident detected, classified, and contained by security team
- 1-4 hours: Internal escalation to CTO and legal team
- 24 hours: Affected customers notified via email if their data is involved
- 72 hours: Regulatory breach notification filed per DPDP Act 2023 requirements
- 30 days: Full post-incident report shared with affected enterprise customers

Data Breach Handling:
- Immediate isolation of affected systems to prevent further exposure
- Forensic investigation initiated within 2 hours
- Customers notified with: nature of breach, data affected, remediation steps
- Dedicated breach response hotline: security@novamed.health
- Cyber insurance policy active (coverage: INR 10 Crore)

Post-Incident:
- Root cause analysis completed within 7 days
- Security controls updated to prevent recurrence
- Annual incident response drills conducted with full team

DATA RESIDENCY:
- All Indian customer data stored in AWS Mumbai (ap-south-1)
- No cross-border data transfer without explicit customer consent
- Data deletion within 30 days of contract termination upon request
